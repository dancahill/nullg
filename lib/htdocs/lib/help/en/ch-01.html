1 - Installation and Configuration

1.1 - Installing NullLogic GroupServer

          To install NullLogic GroupServer, you will need to run 'SETUP.EXE' (Windows) or 'setup' (Linux). If any information is required during this process, you will be prompted for the appropriate information.

    Installing on Linux

          When running the setup script, you may be prompted for the following information:

    • destination install directory
    • SQL server type
    • SQL server's hostname or address
    • SQL server's port
    • SQL database name
    • SQL username
    • SQL password

          When the setup script has completed its task, you will need to initialise the SQL database. In order to do this, you will first need to create the database in your SQL server. This can be done as follows:

      SQL ServerCommand
      MySQLmysqladmin create nullg
      PostgreSQLcreatedb nullg

          The final step to initialising the database is to run the dbutil program. In the case of a new installation, you will need to run dbutil init. Optionally, you can provide a password for the 'administrator' user when running this command. Running dbutil init "visual" will init the database and set the administrator password to visual.

    Installing on Windows

          Installation of the Windows version is considerably simpler. Strictly speaking, you do not need to manually initialise an SQL database. After running the setup wizard, you will be given the option of editing your configuration and/or using dbutil to init, backup, or restore your database. For more information on editing your configuration, see The Configuration Editor.

    Additional Configuration

          At this stage, you should have a fully functional installation of NullLogic GroupServer. There are, however, several different ways of configuring NullLogic GroupServer which may be better suited to its desired task.

          NullLogic GroupServer can be configured to run either as a standalone server or as a CGI with Apache, IIS, or almost any other web server.

          To log in to NullLogic GroupServer for the first time, the username is 'administrator', and the default password is 'visual'. Be sure to change this password as soon as possible.

1.2 - Modes of Operation

          NullLogic GroupServer can be configured to run either as a standalone server or as a CGI with Apache, IIS, or almost any other web server.

          If you intend to run NullLogic GroupServer as a standalone server, you may want to put NullLogic GroupServer in your startup. As an additional security precaution, on Unix/Linux systems, you may want to run NullLogic GroupServer as a non-root user. It is assumed that the server will be run as the user 'nullg'. If the server will be run as another user, care should be taken to ensure adequeate permission is given, especially the etc and filed directories.

          This functionality no longer exists. If you intend to run NullLogic GroupServer as a CGI, simply copy the main NullLogic GroupServer executable (NULLG.EXE on Windows, nullg on Linux) to a directory which your web server recognizes as being a CGI directory, and copy the file nullg.cfg to the same directory. You will also need to copy the directory called nullg from your htdocs directory to your web server's base htdocs directory.

          In either case, you can run multiple installations of NullLogic GroupServer on the same server by creating a different nullg.conf file for each installation. By default, NullLogic GroupServer looks in the current directory first for the configuration file. If it doesn't find one here, it checks ../etc/nullg.conf, and finally, it looks in the default location (/usr/local/nullg/etc/nullg.conf on Linux, and C:\Program Files\NullLogic GroupServer\etc\nullg.conf on Windows).

1.3 - Database Configuration

          Although the default installation uses ODBC to access its database, this configuration is the slowest and least flexible. It is recommended that you at least consider using either MySQL or PostgreSQL instead.

    ODBC (Windows only)

          If your system already has up-to-date MDAC and JET drivers installed, then no further configuration should be necessary. If not, then you may have to download and install the necessary components from Microsoft.

    Microsoft Data Access Components (MDAC) 2.6 RTM
    Microsoft Jet 4.0 Service Pack 3

    MySQL / PostgreSQL

          First, you will need to make an account in your SQL server for NullLogic GroupServer. It is probably wise not to give this account permission to create databases and users, but NullLogic GroupServer should have permission to create new tables.

          The command line tool 'dbutil.exe' can be used to initialise the database for either MySQL or PostgreSQL. Note that the NullLogic GroupServer database must be created (not the tables, just the bare database) prior to running dbutil if you are using MySQL or PostgreSQL. The dbutil program will create and populate all of the necessary tables.

          After you have configured the new database, change the SQL server type in the system configuration to match the appropriate choice.

1.4 - Troubleshooting

          If you are warned when starting the server that you may need to update your MDAC drivers, the you can download the MDAC drivers update directly from Microsoft's web site. You may also need to update your JET drivers.

    Microsoft Data Access Components (MDAC) 2.6 RTM
    Microsoft Jet 4.0 Service Pack 3
