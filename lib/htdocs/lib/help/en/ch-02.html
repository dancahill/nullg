2 - Administration

2.1 - The Configuration Editor

          The configuration editor allows you to modify settings which control the basic operation of the server. Optionally, you could modify the nullg.conf file without the use of the configuration editor. Care should be taken when modifying these settings, as a misconfigured server may be unable to run.

    Server Base Directory - SERVER.DIR.BASE

          The 'Server Base Directory' setting is used to specify the top level directory in which NullLogic GroupServer is installed. Normally, this will be C:\Program Files\NullLogic GroupServer on Windows, and /usr/local/nullg on Linux.

    Server BIN Directory - SERVER.DIR.BIN

          The 'Server BIN Directory' setting is used to specify the directory in which the NullLogic GroupServer binaries are located. Normally, this will be C:\Program Files\NullLogic GroupServer\bin on Windows, and /usr/local/nullg/bin on Linux.

    Server CGI Directory - SERVER.DIR.CGI

          The 'Server CGI Directory' setting is used to specify the directory in which the NullLogic GroupServer CGI (Common Gateway Interface) binaries are located. Normally, this will be C:\Program Files\NullLogic GroupServer\var\share\cgi-bin on Windows, and /usr/local/nullg/var/share/cgi-bin on Linux.

    Server ETC Directory - SERVER.DIR.ETC

          The 'Server ETC Directory' setting is used to specify the directory in which the NullLogic GroupServer log and data files are located. Normally, this will be C:\Program Files\NullLogic GroupServer\etc on Windows, and /usr/local/nullg/etc on Linux.

    Server LIB Directory - SERVER.DIR.LIB

          The 'Server LIB Directory' setting is used to specify the directory in which the NullLogic GroupServer dynamic libraries are located. Normally, this will be C:\Program Files\NullLogic GroupServer\lib on Windows, and /usr/local/nullg/lib on Linux.

    Server VAR Directory - SERVER.DIR.VAR

          The 'Server VAR Directory' setting is used to specify the directory in which the NullLogic GroupServer varying data is located. This includes the database, (SQLite) and database backups, file storage, system logs, mail, and temporary storage space. Normally, this will be C:\Program Files\NullLogic GroupServer\var on Windows, and /usr/local/nullg/var on Linux.

    Server BACKUP Directory - SERVER.DIR.VAR.BACKUP

          The 'Server BACKUP Directory' setting is used to specify the directory in which backups should be stored. Normally, this will be C:\Program Files\NullLogic GroupServer\var\backup on Windows, and /usr/local/nullg/var/backup on Linux.

    Server DB Directory - SERVER.DIR.VAR.DB

          The 'Server DB Directory' setting is used to specify the directory in which SQLite database file is located. Normally, this will be C:\Program Files\NullLogic GroupServer\var\db on Windows, and /usr/local/nullg/var/db on Linux.

    Server File Directory - SERVER.DIR.VAR.FILES

          The 'Server File Directory' setting is used to specify the directory in which the NullLogic GroupServer file bases will be located. Normally, this will be C:\Program Files\NullLogic GroupServer\var\files on Windows, and /usr/local/nullg/var/files on Linux.

    Server Htdocs Directory - SERVER.DIR.VAR.HTDOCS

          The 'Server Htdocs Directory' setting is used to specify the directory in which the NullLogic GroupServer documentation and images are located. Normally, this will be C:\Program Files\NullLogic GroupServer\var\htdocs on Windows, and /usr/local/nullg/var/htdocs on Linux.

    Server Log Directory - SERVER.DIR.VAR.LOG

          The 'Server Log Directory' setting is used to specify the directory in which all log files are stored. Normally, this will be C:\Program Files\NullLogic GroupServer\var\log on Windows, and /usr/local/nullg/var/log on Linux.

    Server Mail Directory - SERVER.DIR.VAR.MAIL

          The 'Server Mail Directory' setting is used to specify the directory in which mailboxes and mail spools are stored. Normally, this will be C:\Program Files\NullLogic GroupServer\var\mail on Windows, and /usr/local/nullg/var/mail on Linux.

    Server Temp Directory - SERVER.DIR.VAR.TMP

          The 'Server Temp Directory' setting is used to specify the directory in which any temporary files will be located. Normally, this will be C:\Program Files\NullLogic GroupServer\var\tmp on Windows, and /usr/local/nullg/var/tmp on Linux.

    Server Log Level - SERVER.LOGLEVEL

          The 'Server Log Level' setting is used to specify the verbosity of the logs which NullLogic GroupServer will keep. The default is '1'.

    Level 0 - Minimal - Only server startups, shutdowns, logins and logouts are logged.
    Level 1 - Light - Logs level 0 events, and notifications when a resource is modified.
    Level 2 - Moderate - Logs level 1 and lower events, each requested resource, and all SQL UPDATE commands.
    Level 3 - Heavy - Logs level 2 and lower events, and all SQL SELECT commands.
    Level 4 - Debug - Logs everything that can be logged.

    Server Username - SERVER.USERNAME

          The 'Server Username' setting is used to specify the user account to which the server should downgrade itself (if run as root). This setting does nothing on Windows.

    HTTP Server Host Address - HTTP.HOSTNAME

          The 'HTTP Server Host Address' setting is used to specify the server name which NullLogic GroupServer will bind itself. This can be the server's fully qualified domain name, the host name, or the IP address of the server. By default, this is set to localhost which will normally only allow requests originating from the same computer the server is installed on.

    HTTP Server Port - HTTP.PORT

          The 'HTTP Server Port' setting is used to specify the TCP/IP port on which NullLogic GroupServer will listen for requests. The default is 4110.

    HTTP Server Max Connections - HTTP.MAXCONN

          The 'HTTP Server Max Connections' setting is used to specify the limit on the number of simultaneous http connections. If this number is set too low, clients may fail to connect to the server. Setting this value too high may result in wasted system resources. The default is 20.

          Note that this setting only affects the maximum number of http connections, and in no way limits the number of users who can log on simultaneously.

    HTTP Server Max Idle Time - HTTP.MAXIDLE

          The 'HTTP Server Max Idle Time' setting controls the amount of idle time (in seconds) which is allowed for an http connection before it is terminated. The default is 120.

    HTTP Server Max POST Size - HTTP.MAXPOSTSIZE

          The 'HTTP Server Max POST Size' setting controls the maximum size (in bytes) of an allowed HTTP POST request. This mainly affects the maximum size of a file which can be uploaded using the web interface.

    SQL Server Type - SQL.TYPE

          The 'SQL Server Type' setting is used to specify whether MySQL, PostgreSQL or ODBC is to be used as the SQL data source. Note that ODBC is currently only available on Windows.

    SQL Server Host Address - SQL.HOSTNAME

          The 'SQL Server Host Address' setting is used to specify the server name on which the MySQL or PostgreSQL server is located. This can be the server's fully qualified domain name, the host name, or the IP address of the server which is hosting the SQL server.

          This setting is not used when ODBC is chosen as a data source.

    SQL Server Port - SQL.PORT

          The 'SQL Server Port' setting is used to specify the TCP/IP port on which the SQL server is listening for requests. The default for PostgreSQL is 5432, and the default for MySQL is 3306.

          This setting is not used when ODBC is chosen as a data source.

    SQL Database Name - SQL.DBNAME

          The 'SQL Database Name' setting is used to specify the name of the database which contains NullLogic GroupServer's data. This should normally be 'nullg' unless another name has been chosen at the time of installation.

    SQL Database Username - SQL.USERNAME

          The 'SQL Database Username' setting is used to specify the user name to use when connecting to the SQL server.

          This setting is not used when ODBC is chosen as a data source.

    SQL Database Password - SQL.PASSWORD

          The 'SQL Database Password' setting is used to specify the password to use when connecting to the SQL server.

          This setting is not used when ODBC is chosen as a data source.

    SQL ODBC Data Source - SQL.ODBC_DSN

          The 'SQL ODBC Data Source' setting is used to specify the Data Source Name (DSN) to use when connecting to the database through ODBC. The default setting should be similar to the following (depending on where NullLogic GroupServer was installed to): 

    	DRIVER={Microsoft Access Driver (*.mdb)};DBQ=C:\database\nullg.mdb

          As an example, if you wanted to use MySQL through ODBC, you would create a System DSN called "MySQL" in the ODBC driver manager and use the following setting to connect to the database: 

    	DRIVER=MySQL;DBQ=MySQL;DATABASE=nullg

          This setting is used only when ODBC is chosen as a data source.

    External File Scan Utility - UTIL.SCANFILE

          The 'External File Scan Utility' setting allows you to specify a script or executable program to use to scan files for viruses and/or inappropriate content. Configuring and using this facility exceeds the scope of this document, however a sample shell script scan_file.sh is included with the source distribution of NullLogic GroupServer.

    External Mail Scan Utility - UTIL.SCANMAIL

          The 'External Mail Scan Utility' setting allows you to specify a script or executable program to use to scan e-mail for viruses and/or inappropriate content. Configuring and using this facility exceeds the scope of this document, however a sample shell script scan_mail.sh is included with the source distribution of NullLogic GroupServer.

2.2 - The Activity Log Viewer

          The access log viewer can be used to view the contents of the access.log file which contains a record of transactions processed by the server. Normally logged activity includes startup and shutdown of the server, logins and logouts, and notifications of record modifications. The detail used when logging events can be modified in the configuration editor by changing the 'Server Log Level' to the desired level.

          Available logging levels include:

    Level 0 - Minimal - Only server startups, shutdowns, logins and logouts are logged.
    Level 1 - Light - Logs level 0 events, and notifications when a record is modified.
    Level 2 - Moderate - Logs level 1 and lower events, each requested resource, and all SQL UPDATE commands.
    Level 3 - Heavy - Logs level 2 and lower events, and all SQL SELECT commands.
    Level 4 - Debug - Logs everything that can be logged.

    The Error Log Viewer

          The error log viewer can be used to view the contents of the error.log file which contains a record of errors caused by the SQL server or NullLogic GroupServer. Some examples and explanations are given below.

          In this example, the client web browser requested a URL which does not translate into an internal command, and it does not represent a file which can be downloaded by the client. 

    	Apr 01 12:05:57 - 127.0.0.1 - Incorrect function call '/nullg/images/logo.png' by administrator

          This, and similar errors usually indicate a problem with the SQL server itself. In this case, the MySQL server which NullLogic GroupServer is trying to connect to is not running. The same error could also be caused if the SQL connection information in the configuration is wrong. Other similar errors may be caused by incorrect SQL query syntax. 

    	Apr 01 12:07:46 - MYSQL error: Can't connect to MySQL server on 'localhost' (10061)

          This particular error may be quite common during initial installation if ODBC is chosen as the SQL type, and you have not installed the necessary updates on your system. For more information, read Troubleshooting. 

    	Apr 01 12:09:16 - ODBC Error: [IM002][Microsoft][ODBC Driver Manager] Data source name not found and no default driver specified (0)

2.4 - The User Editor

          The user editor can be used to add, modify, or delete user accounts. All aspects of a NullLogic GroupServer user's profile, including login information, security access levels, e-mail settings, and personal information can be changed from the user editor.

    Username

          This is the login name which is assigned to the user. Each user must have a unique username.

    Password

          This is the user's login password. Each user should be advised to choose a password which is both easy to remember, and hard to guess. Passwords are encrypted before being stored in the NullLogic GroupServer database. If a user forgets their password, then the administrator will have to assign a new password to the user.

    Group

          This is the primary group of which this user is a member.

    Allow Login

          This option provides a means of disabling the user account.

    Permissions

          Access to records is controlled by a strict system of permissions. At the core of this system is the set of permissions assigned to each user. These permissions control the maximum level of access granted to the user for each section. With the exception of Admin access, permission settings are best demonstrated by the effects of not having them. A user who does not have read permission will not be able to read any records in the relevant section regardless of record ownership or permissions. A user who does not have write permission will not be able to modify, insert or delete any record in that section regardless of record permissions.

          Admin access for a section enables unrestricted access to all records in the relevant section. Admin access also guarantees the user will be able to insert and delete records.

    Administration

          All administrative functions in NullLogic GroupServer require Admin access. Granting Admin access gives the user full permission to all Administration functions. Removing Admin permission, on the other hand, removes access to Administration functions entirely.

    Bookmarks

          This setting controls the access granted to the user for the Bookmarks section.

    Read access allows the user to read bookmarks for which the user has read permissions.
    Modify access allows the user to modify existing bookmarks for which the user has write permissions.
    Insert access allows the user to add new bookmarks.
    Delete access allows the user to delete bookmarks for which the user has write permissions.
    Admin access grants the user unrestricted access to all bookmarks.

    Calendar

          This setting controls the access granted to the user for the Calendar section.

    Read access allows the user to read events for which the user has read permissions.
    Modify access allows the user to modify existing events for which the user has write permissions.
    Insert access allows the user to add new events.
    Delete access allows the user to delete events for which the user has write permissions.
    Admin access grants the user unrestricted access to all events.

    Calls

          This setting controls the access granted to the user for the Calls section.

    Read access allows the user to read calls for which the user has read permissions.
    Modify access allows the user to modify existing calls for which the user has write permissions.
    Insert access allows the user to add new calls.
    Delete access allows the user to delete calls for which the user has write permissions.
    Admin access grants the user unrestricted access to all calls.

    Contacts

          This setting controls the access granted to the user for the Contacts section.

    Read access allows the user to read contacts for which the user has read permissions.
    Modify access allows the user to modify existing contacts for which the user has write permissions.
    Insert access allows the user to add new contacts.
    Delete access allows the user to delete contacts for which the user has write permissions.
    Admin access grants the user unrestricted access to all contacts.

    E-Mail

          This setting controls the access granted to the user for the E-Mail section.

    Read access allows the user to access personal e-mail profiles.
    Modify access allows the user to modify existing personal e-mail profiles.
    Insert access allows the user to add new e-mail profiles.
    Delete access allows the user to delete personal e-mail profiles.

    Files

          This setting controls the access granted to the user for the Files section.

    Read access allows the user to download files for which the user has read permissions.
    Modify access allows the user to modify existing files for which the user has write permissions.
    Insert access allows the user to upload new files.
    Delete access allows the user to delete files for which the user has write permissions.
    Admin access grants the user unrestricted access to all files.

    Forums

          This setting controls the access granted to the user for the Forums section.

    Read access allows the user to read forums and posts.
    Insert access allows the user to post replies to forums.
    Admin access grants the user unrestricted access to all forums.

    Messages

          This setting controls the access granted to the user for the Messages section.

    Read access allows the user to read messages sent to them.
    Insert access allows the user to send messages.
    Delete access allows the user to delete messages sent to them.

    Orders

          This setting controls the access granted to the user for the Orders section.

    Read access allows the user to view existing orders.
    Modify access allows the user to modify existing orders.
    Insert access allows the user to add new orders.
    Delete access allows the user to delete orders.
    Admin access grants the user unrestricted access to all orders as well as products.

    SQL Queries

          This setting controls the access granted to the user for SQL queries in the Searches section.

    Read access allows the user to execute queries for which the user has read permissions.
    Admin access grants the user unrestricted access to all SQL queries.

          Since running arbitrary SQL queries could present a security risk, Admin access is required to add or run new queries.

    Preferences

    Calendar Start

          This defines the starting hour for the calendar day view.

    Calendar Length

          This defines the number of hours to be displayed in the calendar day view.

    Max Results/Page

          This defines the maximum number of results to be listed per page.

    Time Zone

          This defines the time zone adjustment to be applied to all times and dates.

    Geographic Zone

          This defines the geographic zone in which the user works.

    Preferences

    Surname

          This is the surname or last name of the user.

    Given Name

          This is the given name or names of the user.

    Job Title

          This is the job title of the user.

    Division

          This is division the user works in.

    Supervisor

          This is name of the user's supervisor.

    Address

          This is the street address of the user.

    City

          This is the city the user lives in.

    Province/State

          This is the province/state the user lives in.

    Country

          This is the country the user lives in.

    Postal Code

          This is postal code of the user's address.

    Home Phone

          This is the home phone number of the user.

    Work Phone

          This is the work phone number (and extension) of the user.

    FAX

          This is the FAX number of the user.

    Cell Phone

          This is the cell phone number of the user.

    Pager

          This is the pager number of the user.

    E-Mail

          This is the pager number of the user.

    Date of Birth

          This is the user's date of birth.

    Hire Date

          This is the date the user was hired.

    S.I.N. (Social Insurance Number)

          This is the social insurance number (or social security number) of the user.

    Active

          This specifies whether the user is currently active in the company or not.

2.5 - The User Group Editor

          The group editor can be used to add, modify, or delete user groups. Groups can best be defined as logical subcategories of users. The primary use of user groups can be seen in ownership of records such as contacts or events. In this case, a user may choose to grant a higher priveledge to other users in his/her group, and lesser priveledges to users who are not in the same group.

    Group Name

          This is the visible group name used to represent this user group.

    Message of the Day

          This is the message of the day which will be displayed for all users in this group immediately after a successful login.

    Availability

          Clicking on the [Availability] link will take you to a grid which represents one full normal work week, broken up into one hour blocks. Each block with a checkmark indicates an hour which users in this group may mark in their own profile as available time. Any time block which is not selected will be automatically marked 'unavailable' for all users in this group.

2.6 - The Zone Editor

          The zone editor can be used to add, modify, or delete geographical zones. Zoning is an optional feature which allows all users and contacts to be organized into logical subcategories by their geographical location. The primary use of zoning is in the autoassignment feature of the calendar system, where it would be beneficial to automatically assign an event to a user who works in the same region that the contact is located in.

    Zone Name

          This is the visible zone name used to represent this region. When assigning a name to a zone, it is helpful to use a short, but descriptive name of the zone.
