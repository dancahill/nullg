NullLogic Groupware
Chapter 1. Installation and Configuration

Installing NullLogic Groupware

      To install NullLogic Groupware, you will need to run 'SETUP.EXE' (Windows) or 'setup' (Linux). If any information is required during this process, you will be prompted for the appropriate information. Once the setup program is finished, you will need to provide a 'Licence Key' file. This file should be named 'license.txt', and should be placed in the 'etc' subdirectory of NullLogic Groupware. If you do not already have a License Key, an evaluation key can be obtained here.

Installing on Linux

      When running the setup script, you may be prompted for the following information:

  • destination install directory
  • SQL server type
  • SQL server's hostname or address
  • SQL server's port
  • SQL database name
  • SQL username
  • SQL password

      When the setup script has completed its task, you will need to initialise the SQL database. In order to do this, you will first need to create the database in your SQL server. This can be done as follows:

    SQL ServerCommand
    MySQLmysqladmin create groupware
    PostgreSQLcreatedb groupware

      The final step to initialising the database is to run the dbutil program. In the case of a new installation, you will need to run dbutil init. Optionally, you can provide a password for the 'administrator' user when running this command. Running dbutil init "visual" will init the database and set the administrator password to visual.

Installing on Windows

      Installation of the Windows version is considerably simpler. Strictly speaking, you do not need to manually initialise an SQL database. After running the setup wizard, you will be given the option of editing your configuration and/or using dbutil to init, backup, or restore your database. For more information on editing your configuration, see The Configuration Editor.

Additional Configuration

      At this stage, you should have a fully functional installation of NullLogic Groupware. There are, however, several different ways of configuring NullLogic Groupware which may be better suited to its desired task.

      NullLogic Groupware can be configured to run either as a standalone server or as a CGI with Apache, IIS, or almost any other web server.

      To log in to NullLogic Groupware for the first time, the username is 'administrator', and the default password is 'visual'. Be sure to change this password as soon as possible.
