NullLogic Groupware
Chapter 2. Administration

The User Editor

      The user editor can be used to add, modify, or delete user accounts. All aspects of a NullLogic Groupware user's profile, including login information, security access levels, e-mail settings, and personal information can be changed from the user editor.

Username

      This is the login name which is assigned to the user. Each user must have a unique username.

Password

      This is the user's login password. Each user should be advised to choose a password which is both easy to remember, and hard to guess. Passwords are encrypted before being stored in the NullLogic Groupware database. If a user forgets their password, then the administrator will have to assign a new password to the user.

Group

      This is the primary group of which this user is a member.

Allow Login

      This option provides a means of disabling the user account.

Permissions

      Access to records is controlled by a strict system of permissions. At the core of this system is the set of permissions assigned to each user. These permissions control the maximum level of access granted to the user for each section. With the exception of Admin access, permission settings are best demonstrated by the effects of not having them. A user who does not have read permission will not be able to read any records in the relevant section regardless of record ownership or permissions. A user who does not have write permission will not be able to modify, insert or delete any record in that section regardless of record permissions.

      Admin access for a section enables unrestricted access to all records in the relevant section. Admin access also guarantees the user will be able to insert and delete records.

Administration

      All administrative functions in NullLogic Groupware require Admin access. Granting Admin access gives the user full permission to all Administration functions. Removing Admin permission, on the other hand, removes access to Administration functions entirely.

Bookmarks

      This setting controls the access granted to the user for the Bookmarks section.

Read access allows the user to read bookmarks for which the user has read permissions.
Modify access allows the user to modify existing bookmarks for which the user has write permissions.
Insert access allows the user to add new bookmarks.
Delete access allows the user to delete bookmarks for which the user has write permissions.
Admin access grants the user unrestricted access to all bookmarks.

Calendar

      This setting controls the access granted to the user for the Calendar section.

Read access allows the user to read events for which the user has read permissions.
Modify access allows the user to modify existing events for which the user has write permissions.
Insert access allows the user to add new events.
Delete access allows the user to delete events for which the user has write permissions.
Admin access grants the user unrestricted access to all events.

Calls

      This setting controls the access granted to the user for the Calls section.

Read access allows the user to read calls for which the user has read permissions.
Modify access allows the user to modify existing calls for which the user has write permissions.
Insert access allows the user to add new calls.
Delete access allows the user to delete calls for which the user has write permissions.
Admin access grants the user unrestricted access to all calls.

Contacts

      This setting controls the access granted to the user for the Contacts section.

Read access allows the user to read contacts for which the user has read permissions.
Modify access allows the user to modify existing contacts for which the user has write permissions.
Insert access allows the user to add new contacts.
Delete access allows the user to delete contacts for which the user has write permissions.
Admin access grants the user unrestricted access to all contacts.

E-Mail

      This setting controls the access granted to the user for the E-Mail section.

Read access allows the user to access personal e-mail profiles.
Modify access allows the user to modify existing personal e-mail profiles.
Insert access allows the user to add new e-mail profiles.
Delete access allows the user to delete personal e-mail profiles.

Files

      This setting controls the access granted to the user for the Files section.

Read access allows the user to download files for which the user has read permissions.
Modify access allows the user to modify existing files for which the user has write permissions.
Insert access allows the user to upload new files.
Delete access allows the user to delete files for which the user has write permissions.
Admin access grants the user unrestricted access to all files.

Forums

      This setting controls the access granted to the user for the Forums section.

Read access allows the user to read forums and posts.
Insert access allows the user to post replies to forums.
Admin access grants the user unrestricted access to all forums.

Messages

      This setting controls the access granted to the user for the Messages section.

Read access allows the user to read messages sent to them.
Insert access allows the user to send messages.
Delete access allows the user to delete messages sent to them.

Orders

      This setting controls the access granted to the user for the Orders section.

Read access allows the user to view existing orders.
Modify access allows the user to modify existing orders.
Insert access allows the user to add new orders.
Delete access allows the user to delete orders.
Admin access grants the user unrestricted access to all orders as well as products.

SQL Queries

      This setting controls the access granted to the user for SQL queries in the Searches section.

Read access allows the user to execute queries for which the user has read permissions.
Admin access grants the user unrestricted access to all SQL queries.

      Since running arbitrary SQL queries could present a security risk, Admin access is required to add or run new queries.

Preferences

Calendar Start

      This defines the starting hour for the calendar day view.

Calendar Length

      This defines the number of hours to be displayed in the calendar day view.

Max Results/Page

      This defines the maximum number of results to be listed per page.

Time Zone

      This defines the time zone adjustment to be applied to all times and dates.

Geographic Zone

      This defines the geographic zone in which the user works.

Preferences

Surname

      This is the surname or last name of the user.

Given Name

      This is the given name or names of the user.

Job Title

      This is the job title of the user.

Division

      This is division the user works in.

Supervisor

      This is name of the user's supervisor.

Address

      This is the street address of the user.

City

      This is the city the user lives in.

Province/State

      This is the province/state the user lives in.

Country

      This is the country the user lives in.

Postal Code

      This is postal code of the user's address.

Home Phone

      This is the home phone number of the user.

Work Phone

      This is the work phone number (and extension) of the user.

FAX

      This is the FAX number of the user.

Cell Phone

      This is the cell phone number of the user.

Pager

      This is the pager number of the user.

E-Mail

      This is the pager number of the user.

Date of Birth

      This is the user's date of birth.

Hire Date

      This is the date the user was hired.

S.I.N. (Social Insurance Number)

      This is the social insurance number (or social security number) of the user.

Active

      This specifies whether the user is currently active in the company or not.
