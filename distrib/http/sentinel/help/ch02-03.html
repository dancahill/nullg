Sentinel Groupware
Chapter 2. Administration

The Configuration Editor

      The configuration editor allows you to modify settings which control the basic operation of the server. Optionally, you could modify the sentinel.cfg file without the use of the configuration editor. Care should be taken when modifying these settings, as a misconfigured server may be unable to run.

SQL Server Type - SQL_TYPE

      The 'SQL Server Type' setting is used to specify whether MySQL, PostgreSQL or ODBC is to be used as the SQL data source. Note that ODBC is currently only available on Windows.

SQL Server Host Name - SQL_HOSTNAME

      The 'SQL Server Host Name' setting is used to specify the server name on which the MySQL or PostgreSQL server is located. This can be the server's fully qualified domain name, the host name, or the IP address of the server which is hosting the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL Server Port - SQL_PORT

      The 'SQL Server Port' setting is used to specify the TCP/IP port on which the SQL server is listening for requests. The default for PostgreSQL is 5432, and the default for MySQL is 3306.

      This setting is not used when ODBC is chosen as a data source.

SQL Database Name - SQL_DBNAME

      The 'SQL Database Name' setting is used to specify the name of the database which contains Sentinel Groupware's data. This should normally be 'sentinel' unless another name has been chosen at the time of installation.

SQL Database Username - SQL_USERNAME

      The 'SQL Database Username' setting is used to specify the user name to use when connecting to the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL Database Password - SQL_PASSWORD

      The 'SQL Database Password' setting is used to specify the password to use when connecting to the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL ODBC Data Source - SQL_ODBC_DSN

      The 'SQL ODBC Data Source' setting is used to specify the Data Source Name (DSN) to use when connecting to the database through ODBC. The default setting should be similar to the following (depending on where Sentinel Groupware was installed to): 

	DRIVER={Microsoft Access Driver (*.mdb)};DBQ=C:\Program Files\Sentinel Groupware\etc\sentinel.mdb

      As an example, if you wanted to use MySQL through ODBC, you would create a System DSN called "MySQL" in the ODBC driver manager and use the following setting to connect to the database: 

	DRIVER=MySQL;DBQ=MySQL;DATABASE=sentinel

      This setting is used only when ODBC is chosen as a data source.

Sentinel Base Directory - SENTINEL_BASE_DIR

      The 'Sentinel Base Directory' setting is used to specify the top level directory in which Sentinel Groupware was installed. Normally, this will be C:\Program Files\Sentinel Groupware on Windows, and /usr/local/sentinel on Linux.

Sentinel BIN Directory - SENTINEL_BIN_DIR

      The 'Sentinel BIN Directory' setting is used to specify the directory in which the Sentinel Groupware binaries are located. Normally, this will be C:\Program Files\Sentinel Groupware\bin on Windows, and /usr/local/sentinel/bin on Linux.

Sentinel ETC Directory - SENTINEL_ETC_DIR

      The 'Sentinel ETC Directory' setting is used to specify the directory in which the Sentinel Groupware log and data files are located. Normally, this will be C:\Program Files\Sentinel Groupware\etc on Windows, and /usr/local/sentinel/etc on Linux.

Sentinel File Directory - SENTINEL_FILE_DIR

      The 'Sentinel File Directory' setting is used to specify the directory in which the Sentinel Groupware file bases will be located. Normally, this will be C:\Program Files\Sentinel Groupware\files on Windows, and /usr/local/sentinel/files on Linux.

Sentinel HTTP Directory - SENTINEL_HTTP_DIR

      The 'Sentinel HTTP Directory' setting is used to specify the directory in which the Sentinel Groupware documentation and images are located. Normally, this will be C:\Program Files\Sentinel Groupware\http on Windows, and /usr/local/sentinel/http on Linux.

Sentinel Log Level - SENTINEL_LOGLEVEL

      The 'Sentinel Log Level' setting is used to specify the verbosity of the logs which Sentinel Groupware will keep. The default is '1'.

Level 0 - Minimal - Only server startups, shutdowns, logins and logouts are logged.
Level 1 - Light - Logs level 0 events, and notifications when a resource is modified.
Level 2 - Moderate - Logs level 1 and lower events, each requested resource, and all UPDATE commands.
Level 3 - Heavy - Logs level 2 and lower events, and all SELECT commands.
Level 4 - Debug - Logs everything that can be logged.

Sentinel Host Name - SENTINEL_HOSTNAME

      The 'Sentinel Host Name' setting is used to specify the server name which the Sentinel Groupware server will bind itself. This can be the server's fully qualified domain name, the host name, or the IP address of the server. By default, this is set to localhost which will normally only allow requests originating from the same computer the server is installed on.

Sentinel Port - SENTINEL_PORT

      The 'Sentinel Port' setting is used to specify the TCP/IP port on which Sentinel Groupware will listen for requests. The default is 4110.

Sentinel Max Connections - SENTINEL_MAXCONN

      The 'Sentinel Max Connections' setting is used to specify the limit on the number of simultaneous http connections. If this number is set too low, clients may fail to connect to the server. Setting this value too high may result in wasted system resources. The default is 20.

      Note that this setting only affects the maximum number of http connections, and in no way limits the number of users who can log on simultaneously.

Sentinel Max Idle Time - SENTINEL_MAXIDLE

      The 'Sentinel Max Idle Time' setting controls the amount of idle time (in seconds) which is allowed for an http connection before it is terminated. The default is 120.
