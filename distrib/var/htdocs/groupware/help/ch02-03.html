NullLogic Groupware
Chapter 2. Administration

The Configuration Editor

      The configuration editor allows you to modify settings which control the basic operation of the server. Optionally, you could modify the groupware.cfg file without the use of the configuration editor. Care should be taken when modifying these settings, as a misconfigured server may be unable to run.

Server Base Directory - SERVER_BASE_DIR

      The 'Server Base Directory' setting is used to specify the top level directory in which NullLogic Groupware was installed. Normally, this will be C:\Program Files\NullLogic Groupware on Windows, and /usr/local/groupware on Linux.

Server BIN Directory - SERVER_BIN_DIR

      The 'Server BIN Directory' setting is used to specify the directory in which the NullLogic Groupware binaries are located. Normally, this will be C:\Program Files\NullLogic Groupware\bin on Windows, and /usr/local/groupware/bin on Linux.

Server CGI Directory - SERVER_CGI_DIR

      The 'Server CGI Directory' setting is used to specify the directory in which the NullLogic Groupware CGI (Common Gateway Interface) binaries are located. Normally, this will be C:\Program Files\NullLogic Groupware\cgi-bin on Windows, and /usr/local/groupware/cgi-bin on Linux.

Server ETC Directory - SERVER_ETC_DIR

      The 'Server ETC Directory' setting is used to specify the directory in which the NullLogic Groupware log and data files are located. Normally, this will be C:\Program Files\NullLogic Groupware\etc on Windows, and /usr/local/groupware/etc on Linux.

Server File Directory - SERVER_FILE_DIR

      The 'Server File Directory' setting is used to specify the directory in which the NullLogic Groupware file bases will be located. Normally, this will be C:\Program Files\NullLogic Groupware\files on Windows, and /usr/local/groupware/files on Linux.

Server Htdocs Directory - SERVER_HTDOCS_DIR

      The 'Server Htdocs Directory' setting is used to specify the directory in which the NullLogic Groupware documentation and images are located. Normally, this will be C:\Program Files\NullLogic Groupware\htdocs on Windows, and /usr/local/groupware/htdocs on Linux.

Server Log Level - SERVER_LOGLEVEL

      The 'Server Log Level' setting is used to specify the verbosity of the logs which NullLogic Groupware will keep. The default is '1'.

Level 0 - Minimal - Only server startups, shutdowns, logins and logouts are logged.
Level 1 - Light - Logs level 0 events, and notifications when a resource is modified.
Level 2 - Moderate - Logs level 1 and lower events, each requested resource, and all SQL UPDATE commands.
Level 3 - Heavy - Logs level 2 and lower events, and all SQL SELECT commands.
Level 4 - Debug - Logs everything that can be logged.

Server Host Address - SERVER_HOSTNAME

      The 'Server Host Address' setting is used to specify the server name which the NullLogic Groupware server will bind itself. This can be the server's fully qualified domain name, the host name, or the IP address of the server. By default, this is set to localhost which will normally only allow requests originating from the same computer the server is installed on.

Server Port - SERVER_PORT

      The 'Server Port' setting is used to specify the TCP/IP port on which NullLogic Groupware will listen for requests. The default is 4110.

Server Max Connections - SERVER_MAXCONN

      The 'Server Max Connections' setting is used to specify the limit on the number of simultaneous http connections. If this number is set too low, clients may fail to connect to the server. Setting this value too high may result in wasted system resources. The default is 20.

      Note that this setting only affects the maximum number of http connections, and in no way limits the number of users who can log on simultaneously.

Server Max Idle Time - SERVER_MAXIDLE

      The 'Server Max Idle Time' setting controls the amount of idle time (in seconds) which is allowed for an http connection before it is terminated. The default is 120.

SQL Server Type - SQL_TYPE

      The 'SQL Server Type' setting is used to specify whether MySQL, PostgreSQL or ODBC is to be used as the SQL data source. Note that ODBC is currently only available on Windows.

SQL Server Host Address - SQL_HOSTNAME

      The 'SQL Server Host Address' setting is used to specify the server name on which the MySQL or PostgreSQL server is located. This can be the server's fully qualified domain name, the host name, or the IP address of the server which is hosting the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL Server Port - SQL_PORT

      The 'SQL Server Port' setting is used to specify the TCP/IP port on which the SQL server is listening for requests. The default for PostgreSQL is 5432, and the default for MySQL is 3306.

      This setting is not used when ODBC is chosen as a data source.

SQL Database Name - SQL_DBNAME

      The 'SQL Database Name' setting is used to specify the name of the database which contains NullLogic Groupware's data. This should normally be 'groupware' unless another name has been chosen at the time of installation.

SQL Database Username - SQL_USERNAME

      The 'SQL Database Username' setting is used to specify the user name to use when connecting to the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL Database Password - SQL_PASSWORD

      The 'SQL Database Password' setting is used to specify the password to use when connecting to the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL ODBC Data Source - SQL_ODBC_DSN

      The 'SQL ODBC Data Source' setting is used to specify the Data Source Name (DSN) to use when connecting to the database through ODBC. The default setting should be similar to the following (depending on where NullLogic Groupware was installed to): 

	DRIVER={Microsoft Access Driver (*.mdb)};DBQ=C:\Program Files\NullLogic Groupware\var\db\groupware.mdb

      As an example, if you wanted to use MySQL through ODBC, you would create a System DSN called "MySQL" in the ODBC driver manager and use the following setting to connect to the database: 

	DRIVER=MySQL;DBQ=MySQL;DATABASE=groupware

      This setting is used only when ODBC is chosen as a data source.
