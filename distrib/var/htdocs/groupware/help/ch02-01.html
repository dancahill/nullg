Chapter 2. Administration

The Access Log Viewer

      The access log viewer can be used to view the contents of the access.log file which contains a record of transactions processed by the server. Normally logged activity includes startup and shutdown of the server, logins and logouts, and notifications of record modifications. The detail used when logging events can be modified in the configuration editor by changing the 'Server Log Level' to the desired level.

      Available logging levels include:

Level 0 - Minimal - Only server startups, shutdowns, logins and logouts are logged.
Level 1 - Light - Logs level 0 events, and notifications when a record is modified.
Level 2 - Moderate - Logs level 1 and lower events, each requested resource, and all SQL UPDATE commands.
Level 3 - Heavy - Logs level 2 and lower events, and all SQL SELECT commands.
Level 4 - Debug - Logs everything that can be logged.
