NullLogic Groupware
Chapter 2. Administration

The Configuration Editor

      The configuration editor allows you to modify settings which control the basic operation of the server. Optionally, you could modify the groupware.cfg file without the use of the configuration editor. Care should be taken when modifying these settings, as a misconfigured server may be unable to run.

Server Base Directory - SERVER.DIR.BASE

      The 'Server Base Directory' setting is used to specify the top level directory in which NullLogic Groupware is installed. Normally, this will be C:\Program Files\NullLogic Groupware on Windows, and /usr/local/nullgroupware on Linux.

Server BIN Directory - SERVER.DIR.BIN

      The 'Server BIN Directory' setting is used to specify the directory in which the NullLogic Groupware binaries are located. Normally, this will be C:\Program Files\NullLogic Groupware\bin on Windows, and /usr/local/nullgroupware/bin on Linux.

Server CGI Directory - SERVER.DIR.CGI

      The 'Server CGI Directory' setting is used to specify the directory in which the NullLogic Groupware CGI (Common Gateway Interface) binaries are located. Normally, this will be C:\Program Files\NullLogic Groupware\cgi-bin on Windows, and /usr/local/nullgroupware/cgi-bin on Linux.

Server ETC Directory - SERVER.DIR.ETC

      The 'Server ETC Directory' setting is used to specify the directory in which the NullLogic Groupware log and data files are located. Normally, this will be C:\Program Files\NullLogic Groupware\etc on Windows, and /usr/local/nullgroupware/etc on Linux.

Server LIB Directory - SERVER.DIR.LIB

      The 'Server LIB Directory' setting is used to specify the directory in which the NullLogic Groupware dynamic libraries are located. Normally, this will be C:\Program Files\NullLogic Groupware\lib on Windows, and /usr/local/nullgroupware/lib on Linux.

Server VAR Directory - SERVER.DIR.VAR

      The 'Server VAR Directory' setting is used to specify the directory in which the NullLogic Groupware varying data is located. This includes the database, (SQLite) and database backups, file storage, system logs, mail, and temporary storage space. Normally, this will be C:\Program Files\NullLogic Groupware\var on Windows, and /usr/local/nullgroupware/var on Linux.

Server BACKUP Directory - SERVER.DIR.VAR.BACKUP

      The 'Server BACKUP Directory' setting is used to specify the directory in which backups should be stored. Normally, this will be C:\Program Files\NullLogic Groupware\var\backup on Windows, and /usr/local/nullgroupware/var/backup on Linux.

Server DB Directory - SERVER.DIR.VAR.DB

      The 'Server DB Directory' setting is used to specify the directory in which SQLite database file is located. Normally, this will be C:\Program Files\NullLogic Groupware\var\db on Windows, and /usr/local/nullgroupware/var/db on Linux.

Server File Directory - SERVER.DIR.VAR.FILES

      The 'Server File Directory' setting is used to specify the directory in which the NullLogic Groupware file bases will be located. Normally, this will be C:\Program Files\NullLogic Groupware\var\files on Windows, and /usr/local/nullgroupware/var/files on Linux.

Server Htdocs Directory - SERVER.DIR.VAR.HTDOCS

      The 'Server Htdocs Directory' setting is used to specify the directory in which the NullLogic Groupware documentation and images are located. Normally, this will be C:\Program Files\NullLogic Groupware\var\htdocs on Windows, and /usr/local/nullgroupware/var/htdocs on Linux.

Server Log Directory - SERVER.DIR.VAR.LOG

      The 'Server Log Directory' setting is used to specify the directory in which all log files are stored. Normally, this will be C:\Program Files\NullLogic Groupware\var\log on Windows, and /usr/local/nullgroupware/var/log on Linux.

Server Mail Directory - SERVER.DIR.VAR.MAIL

      The 'Server Mail Directory' setting is used to specify the directory in which mailboxes and mail spools are stored. Normally, this will be C:\Program Files\NullLogic Groupware\var\mail on Windows, and /usr/local/nullgroupware/var/mail on Linux.

Server Temp Directory - SERVER.DIR.VAR.TMP

      The 'Server Temp Directory' setting is used to specify the directory in which any temporary files will be located. Normally, this will be C:\Program Files\NullLogic Groupware\var\tmp on Windows, and /usr/local/nullgroupware/var/tmp on Linux.

Server Log Level - SERVER.LOGLEVEL

      The 'Server Log Level' setting is used to specify the verbosity of the logs which NullLogic Groupware will keep. The default is '1'.

Level 0 - Minimal - Only server startups, shutdowns, logins and logouts are logged.
Level 1 - Light - Logs level 0 events, and notifications when a resource is modified.
Level 2 - Moderate - Logs level 1 and lower events, each requested resource, and all SQL UPDATE commands.
Level 3 - Heavy - Logs level 2 and lower events, and all SQL SELECT commands.
Level 4 - Debug - Logs everything that can be logged.

Server Username - SERVER.USERNAME

      The 'Server Username' setting is used to specify the user account to which the server should downgrade itself (if run as root). This setting does nothing on Windows.

HTTP Server Host Address - HTTP.HOSTNAME

      The 'HTTP Server Host Address' setting is used to specify the server name which the NullLogic Groupware server will bind itself. This can be the server's fully qualified domain name, the host name, or the IP address of the server. By default, this is set to localhost which will normally only allow requests originating from the same computer the server is installed on.

HTTP Server Port - HTTP.PORT

      The 'HTTP Server Port' setting is used to specify the TCP/IP port on which NullLogic Groupware will listen for requests. The default is 4110.

HTTP Server Max Connections - HTTP.MAXCONN

      The 'HTTP Server Max Connections' setting is used to specify the limit on the number of simultaneous http connections. If this number is set too low, clients may fail to connect to the server. Setting this value too high may result in wasted system resources. The default is 20.

      Note that this setting only affects the maximum number of http connections, and in no way limits the number of users who can log on simultaneously.

HTTP Server Max Idle Time - HTTP.MAXIDLE

      The 'HTTP Server Max Idle Time' setting controls the amount of idle time (in seconds) which is allowed for an http connection before it is terminated. The default is 120.

SQL Server Type - SQL.TYPE

      The 'SQL Server Type' setting is used to specify whether MySQL, PostgreSQL or ODBC is to be used as the SQL data source. Note that ODBC is currently only available on Windows.

SQL Server Host Address - SQL.HOSTNAME

      The 'SQL Server Host Address' setting is used to specify the server name on which the MySQL or PostgreSQL server is located. This can be the server's fully qualified domain name, the host name, or the IP address of the server which is hosting the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL Server Port - SQL.PORT

      The 'SQL Server Port' setting is used to specify the TCP/IP port on which the SQL server is listening for requests. The default for PostgreSQL is 5432, and the default for MySQL is 3306.

      This setting is not used when ODBC is chosen as a data source.

SQL Database Name - SQL.DBNAME

      The 'SQL Database Name' setting is used to specify the name of the database which contains NullLogic Groupware's data. This should normally be 'groupware' unless another name has been chosen at the time of installation.

SQL Database Username - SQL.USERNAME

      The 'SQL Database Username' setting is used to specify the user name to use when connecting to the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL Database Password - SQL.PASSWORD

      The 'SQL Database Password' setting is used to specify the password to use when connecting to the SQL server.

      This setting is not used when ODBC is chosen as a data source.

SQL ODBC Data Source - SQL.ODBC_DSN

      The 'SQL ODBC Data Source' setting is used to specify the Data Source Name (DSN) to use when connecting to the database through ODBC. The default setting should be similar to the following (depending on where NullLogic Groupware was installed to): 

	DRIVER={Microsoft Access Driver (*.mdb)};DBQ=C:\Program Files\NullLogic Groupware\var\db\groupware.mdb

      As an example, if you wanted to use MySQL through ODBC, you would create a System DSN called "MySQL" in the ODBC driver manager and use the following setting to connect to the database: 

	DRIVER=MySQL;DBQ=MySQL;DATABASE=groupware

      This setting is used only when ODBC is chosen as a data source.
