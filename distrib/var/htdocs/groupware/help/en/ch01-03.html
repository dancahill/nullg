NullLogic Groupware
Chapter 1. Installation and Configuration

Database Configuration

      Although the default installation uses ODBC to access its database, this configuration is the slowest and least flexible. It is recommended that you at least consider using either MySQL or PostgreSQL instead.

ODBC (Windows only)

      If your system already has up-to-date MDAC and JET drivers installed, then no further configuration should be necessary. If not, then you may have to download and install the necessary components from Microsoft.

Microsoft Data Access Components (MDAC) 2.6 RTM
Microsoft Jet 4.0 Service Pack 3

MySQL / PostgreSQL

      First, you will need to make an account in your SQL server for NullLogic Groupware. It is probably wise not to give this account permission to create databases and users, but NullLogic Groupware should have permission to create new tables.

      The command line tool 'dbutil.exe' can be used to initialise the database for either MySQL or PostgreSQL. Note that the NullLogic Groupware database must be created (not the tables, just the bare database) prior to running dbutil if you are using MySQL or PostgreSQL. The dbutil program will create and populate all of the necessary tables.

      After you have configured the new database, change the SQL server type in the system configuration to match the appropriate choice.
