NullLogic Groupware
Chapter 1. Installation and Configuration

Modes of Operation

      NullLogic Groupware can be configured to run either as a standalone server or as a CGI with Apache, IIS, or almost any other web server.

      If you intend to run NullLogic Groupware as a standalone server, you may want to put the NullLogic Groupware server in your startup. As an additional security precaution, on Unix/Linux systems, you may want to run NullLogic Groupware as a non-root user. It is assumed that the server will be run as the user 'groupware'. If the server will be run as another user, care should be taken to ensure adequeate permission is given, especially the etc and filed directories.

      If you intend to run NullLogic Groupware as a CGI, simply copy the main NullLogic Groupware executable (GROUPWARE.EXE on Windows, groupware on Linux) to a directory which your web server recognizes as being a CGI directory, and copy the file groupware.cfg to the same directory. You will also need to copy the directory called groupware from your htdocs directory to your web server's base htdocs directory.

      In either case, you can run multiple installations of NullLogic Groupware on the same server by creating a different groupware.cfg file for each installation. By default, the NullLogic Groupware server looks in the current directory first for the configuration file. If it doesn't find one here, it checks ../etc/groupware.cfg, and finally, it looks in the default location (/usr/local/nullgroupware/etc/groupware.cfg on Linux, and C:\Program Files\NullLogic Groupware\etc\groupware.cfg on Windows).
